1. Запуск приложения
Откройте терминал или командную строку.

- Перейдите в папку с приложением:
cd путь/к/папке/с/приложением

- Запустите приложение:
streamlit run app.py
После запуска в консоли появится ссылка (обычно http://localhost:8501) — откройте её в браузере.

2. Интерфейс приложения
Приложение состоит из нескольких секций:
- Боковая панель (слева) — фильтры и настройки.
- Основная область (справа) — графики и таблицы с результатами.

3. Работа с фильтрами (боковая панель)

Шаг 1. Выбор городов
В выпадающем списке «Выберите города для анализа» отметьте один или несколько городов.
Можно выбрать все города или только интересующие вас.

Шаг 2. Выбор временного периода
Используйте виджет выбора даты, чтобы задать:
- начальную дату периода;
- конечную дату периода.
По умолчанию отображаются все доступные данные.

Шаг 3. Выбор метрики для сравнения
В выпадающем списке «Выберите метрику для сравнения» выберите показатель:
- avg_temp — средняя температура;
- total_precip — общее количество осадков;
- avg_wind — средняя скорость ветра.

Шаг 4. Выбор метрики для временного ряда
В выпадающем списке «Выберите метрику для временного ряда» выберите показатель для построения графика динамики:
- avg_temp — средняя температура;
- total_precip — общее количество осадков;
- avg_wind — средняя скорость ветра.

4. Анализ данных (основная область)

Секция 1. Сравнение по городам
Отображается столбчатая диаграмма, показывающая среднее значение выбранной метрики по городам.
Заголовок графика отражает выбранную метрику.
Ось X — города, ось Y — значение метрики.

Секция 2. Временные ряды и прогноз
Для каждого выбранного города строится график динамики выбранной метрики.

На графике отображаются:
- Реальные данные — сплошная линия.
- Скользящее среднее (за 7 дней) — пунктирная линия.

График позволяет увидеть:
- общую тенденцию изменения показателя;
- сезонные колебания;
- аномальные значения.

Секция 3. Статистика по категориям
Три круговые диаграммы, показывающие распределение данных по категориям:
- «Распределение температур» — доли дней с холодной, умеренной и жаркой погодой.
- «Распределение осадков» — доли дней без осадков, с небольшими и сильными осадками.
- «Уровень комфортности» — доли комфортных, умеренно комфортных и некомфортных дней.

Секция 4. Дополнительная аналитика
Корреляционная матрица — таблица с коэффициентами корреляции между показателями (avg_temp, total_precip, avg_wind).
Значения от −1 до 1 показывают силу и направление связи между переменными.

5. Интерпретация результатов

Как читать графики:
Столбчатые диаграммы: высота столбца соответствует среднему значению метрики для города.

Графики временных рядов:
- резкий рост/падение — возможные аномалии;
- плавный тренд — устойчивая тенденция;
- сезонность — повторяющиеся паттерны в определённые периоды.

Круговые диаграммы: размер сектора показывает долю соответствующей категории.

Корреляционная матрица:
- близкие к 1 — сильная положительная связь;
- близкие к −1 — сильная отрицательная связь;
- около 0 — отсутствие связи.

6. Практические примеры использования

Пример 1. Сравнение климата городов
1) Выберите 2–3 города.
2) Установите интересующий период (например, лето 2023 года).
3) Выберите метрику avg_temp.
4) Проанализируйте различия в средних температурах.

Пример 2. Анализ осадков
1) Выберите город.
2) Задайте период (например, год).
3) Выберите метрику total_precip.
4) Изучите график временных рядов на предмет аномальных осадков.

7. Советы по работе с приложением

Для детального анализа: сужайте временной период — так легче увидеть детали.
Для общего обзора: выбирайте большие периоды (год, несколько лет).
При сравнении городов: выбирайте одинаковые периоды для всех городов.
Если графики не отображаются: проверьте, выбраны ли города и период.
При ошибках: перезапустите приложение и убедитесь, что данные загружены корректно.

8. Устранение неполадок
Проблема: графики не отображаются.
Решение:
- проверьте, выбраны ли города в фильтре;
- убедитесь, что выбранный период содержит данные;
- перезапустите приложение.

Проблема: ошибка загрузки данных.
Решение:
- проверьте целостность файла с данными;
- убедитесь, что файл находится в правильной папке;
- проверьте формат данных (должны быть колонки city, date, avg_temp, total_precip, avg_wind).

Проблема: некорректные значения на графиках.
Решение:
- проверьте фильтры — возможно, выбран слишком большой период;
- проанализируйте исходные данные на наличие аномалий.
